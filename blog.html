Important FAQ:

What is the Document Object Model (DOM)?

The Document Object Model (DOM) is like a map of a web page that helps your browser understand and display the content. It turns everything on a webpage (like text, images, buttons, etc.) into objects that JavaScript can interact with.

How do you select an element from the DOM?

To select an element from the DOM in JavaScript, you use methods like getElementById, getElementsByClassName, querySelector, or querySelectorAll

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM that allows you to handle events more efficiently, especially when dealing with multiple elements. Instead of attaching an event listener to each individual element, you attach a single event listener to a parent element, and then let the event "bubble up" to that parent. This way, you manage all child elements from one place.


Improves performance: Instead of adding event listeners to every single child element (which can slow down your page), you only need one listener on the parent. Handles dynamically added elements: If new elements are added to the DOM after your page loads, event delegation still works. You dont need to add new listeners for those elements because the parent is still listening. Cleaner code: Fewer event listeners mean less code to maintain, making your code easier to read and manage.
